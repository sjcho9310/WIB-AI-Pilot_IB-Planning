STEP B — 트랜치별 반복 입력
투자유형 ↓ 상품유형 ↓ Book유형 당사주선/투자 트랜치 당사주선/인수규모(억원) 당사투자금액(억원) 투자수익률구분 + 투자수익률(%) 투자기간만기(월) + BRR등급 선취수수료구분 + 금액(억원)
3단 연동 드롭다운 — 핵심 설계 포인트

상품유형 → 허용 Book유형 목록 + 수익률구분 교체
Book유형 + 투자수익률구분
상위 항목 변경 시 → 하위 항목 범위 밖이면 첫 번째 값으로 자동 리셋
①
당사투자금액 = 0이면 투자수익률구분·투자수익률 입력 비활성화
→ 주선/인수 전용 딜에서 수익률 오입력 방지
②
복합 Deal 트랜치 = [단일순위 / 선순위 / 중후순위 / EBL / Mezzanine / Equity] 멀티셀렉트
→ 선택순서대로 트랜치 1·2·3 배정, 트랜치 타입 필드 자동 고정

핵심 자동 산출 로직
FTP 원가율 자동 계산
Book ∈ {채무보증, Book미사용, 그룹펀드} → 미적용
Book = 종금Book → 종금Book FTP 테이블 사용
나머지 → IB Book FTP 테이블 사용
선형보간: numpy.interp(만기월, tenor구간, 금리배열)
Carry손익 산출 공식
보유월수 = min(투자기간만기, 기표일~12/31 잔여월수)
이자수익 = 투자금액 × 수익률% / 100 × 보유월수 / 12
자본원가 = 투자금액 × FTP% / 100 × 보유월수 / 12
Carry손익 = 이자수익 − 자본원가
